## 简介
本故障排除指南旨在帮助开发人员和系统管理员快速诊断和解决AI视频分析系统中遇到的常见问题。该系统由前端H5应用和后端Node.js服务组成，支持视频文件上传、内容分析、融合分析和音乐提示词生成等功能。

系统架构采用前后端分离设计，前端使用Vue 3框架，后端使用Express框架提供RESTful API和WebSocket实时通信。文件上传功能支持批量上传和实时进度跟踪，AI分析功能通过调用阿里云DashScope API实现视频理解和内容生成。

## 常见问题与解决方案

### 1. Windows系统下视频文件路径错误
**问题描述：**
- 真实的视频文件被错误识别为"不是视频文件"
- 错误信息：`The file: /path/to/video.mp4 is not exists!`
- 系统报告文件不存在，但文件确实存在且可播放

**根本原因：**
阿里云DashScope API对Windows系统下的file://协议路径格式有特殊要求。错误的路径格式导致API无法访问本地文件。

**解决方案：**
```python
# 错误格式（三个斜杠）
file:///E:/path/to/video.mp4

# 正确格式（两个斜杠）
file://E:/path/to/video.mp4
```

**修复位置：**
- `backend/scripts/video_analyzer.py` 第116-120行
- `backend/scripts/video_analyzer.py` 第271-275行

**验证方法：**
```bash
cd backend
python scripts/video_analyzer.py --video-path "upload/personal/test.mp4" --type content
```

### 4. API调用失败处理
**问题描述：**
- HTTP API调用失败，错误400：`InvalidParameter: The provided URL does not appear to be valid`
- 系统自动回退到Python SDK

**解决方案：**
系统已实现智能回退机制：
1. 优先尝试Python SDK（支持本地文件）
2. SDK失败时回退到HTTP API

### 5. 文件上传验证失败
**问题描述：**
- 前端上传文件时返回`VALIDATION_ERROR`
- 文件格式、大小或数量不符合要求

**根本原因：**
后端对上传文件进行了严格的验证，包括文件格式、大小、数量和类型一致性。

**解决方案：**
1. 确保文件格式为MP4或AVI
2. 单个文件大小不超过300MB
3. 每次上传不超过3个文件
4. 同一批次上传的文件必须是相同格式

**验证代码：**
```javascript
// 后端文件验证逻辑
const fileValidationSchema = Joi.object({
  files: Joi.array().items(
    Joi.object({
      name: Joi.string().required(),
      size: Joi.number().integer().min(1).max(config.maxFileSize).required(),
      type: Joi.string().allow('').optional()
    })
  ).min(1).max(3).required()
});
```

### 6. WebSocket连接问题
**问题描述：**
- 前端无法接收实时上传进度
- 控制台显示WebSocket连接失败

**根本原因：**
前端WebSocket连接配置不正确或后端WebSocket服务未正常启动。

**解决方案：**
1. 确保后端服务在端口8005运行
2. 检查前端WebSocket连接配置：
```javascript
const socket = io(`http://localhost:8005`, {
  transports: ['websocket', 'polling'],
  forceNew: true,
  timeout: 8000,
  reconnection: true,
  reconnectionAttempts: 5,
  reconnectionDelay: 1000,
  reconnectionDelayMax: 2000
})
```

3. 验证WebSocket连接状态：
```javascript
socket.on('connect', () => {
  console.log('WebSocket connected successfully')
})

socket.on('connect_error', (error) => {
  console.error('WebSocket connection error:', error)
})
```

### 7. 路径遍历攻击防护
**问题描述：**
- 上传文件时返回"非法的文件路径"错误
- 文件上传被阻止

**根本原因：**
系统实现了路径遍历攻击防护，确保文件只能保存在指定的上传目录内。

**解决方案：**
1. 确保文件路径解析正确：
```javascript
const resolvedPath = path.resolve(inputPath);
const uploadBaseDir = path.resolve(config.uploadBaseDir || path.join(__dirname, '../../upload'));

// 验证文件路径是否在允许的上传目录内
if (!resolvedPath.startsWith(uploadBaseDir)) {
  return res.status(400).json({
    success: false,
    error: '非法的文件路径，请使用有效的上传文件'
  });
}
```

2. 使用正确的文件路径格式：
```javascript
// 正确的路径格式
const videoUrl = `${req.protocol}://${req.get('host')}/uploads/${relativePath}`;
```

### 8. AI分析超时问题
**问题描述：**
- 视频分析请求超时
- 前端显示"分析超时，请重试或联系管理员"

**根本原因：**
AI分析服务设置了超时限制，大文件或复杂分析可能超过超时时间。

**解决方案：**
1. 增加前端超时时间：
```javascript
// 前端超时控制 (3分钟)
const controller = new AbortController()
const timeoutId = setTimeout(() => {
  controller.abort()
}, 180000) // 3分钟超时
```

2. 优化后端超时设置：
```javascript
// 后端超时设置
this.timeout = 120000; // 120秒超时
```

3. 压缩大视频文件：
```bash
# 使用FFmpeg压缩
ffmpeg -i input.mp4 -b:v 1M -b:a 128k output.mp4
```

## 紧急恢复程序

### 完全重置
1. 停止所有服务：
```bash
# 停止后端
pkill -f "node.*src/app.js"

# 停止前端
pkill -f "npm run dev"
```

2. 清理临时文件：
```bash
# 清理上传目录（谨慎操作）
# rm -rf backend/upload/*
```

3. 重新安装依赖：
```bash
cd backend
rm -rf node_modules package-lock.json
npm install

cd ../frontend
rm -rf node_modules package-lock.json
npm install
```

4. 重新启动服务：
```bash
cd backend
npm start

cd ../frontend
npm run dev
```

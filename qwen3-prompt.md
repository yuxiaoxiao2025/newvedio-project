提示（Prompt）是您输入给大模型（LLM）的文本信息，用于明确地告诉模型您想要解决的问题或完成的任务，也是大语言模型理解用户需求并生成相关、准确回答或内容的基础。为了帮助您更高效地使用 LLM，本教程提供一系列实用技巧，帮助您设计与优化 Prompt。

## 设计 Prompt

### 构建清晰明确的 Prompt

- 原则：任务描述越清晰、具体、无歧义，输出越符合期望。
- 要点：明确目的、背景、风格、语气、受众与输出形式。

| 模糊的 Prompt | 清晰具体的 Prompt |
| --- | --- |
| 我想推广公司的新产品。我的公司名为阿里云百炼，新产品名为 Zephyr Z9，是一款轻薄便携的手机。帮我创建一条微博帖子。 | 请为我司“阿里云百炼”最新推出的“Zephyr Z9”轻薄便携手机设计一条吸引眼球的微博推广帖。内容需彰显 Zephyr Z9 的卖点（轻薄设计、高性能、便利性），并融入创意元素提升互动。贴文需简洁有力、符合平台风格与字数限制，鼓励读者了解更多信息或购买。 |
| 你是一位资深 PHP 编程专家，现在接到一个明确的开发任务。诉求是：${require}。请使用 PHP 语言详细描述一种实现该诉求的算法或方法，并考虑边界情况与错误处理，以及如何通过高效且安全的代码来满足此需求。 | 作为一位经验丰富的 PHP 编程专家，你的任务为：${require}。请详细阐述一种利用 PHP 实现该需求的高效方法，覆盖：实现步骤（函数、类、数据结构）、边界条件分析、错误处理机制、安全性考量与性能优化建议；要求条理清晰、便于复用。 |

- 提示优化工具：可以在 Prompt 页面使用“自动优化”对提示进行扩写与细节补充（不涉及外链）。

### 使用 Prompt 框架

为系统化提供背景与目的，建议使用“Prompt 框架”规范化结构：

- 背景：与任务紧密相关的背景信息，确保语境一致。
- 目的：期望完成的具体任务与精确目标。
- 风格：输出写作风格（人物/流派/专家类型）。
- 语气：正式、诙谐、温馨等，匹配使用场景。
- 受众：面向群体（专业人士、入门者、儿童等）。
- 输出：结果形式（列表、JSON、报告等）。

| 项 | 未使用框架 | 使用框架 |
| --- | --- | --- |
| Prompt | 我想推广公司的新产品。帮我创建一条微博帖子，简洁而深具影响力。 | #背景# 我司阿里云百炼新品 Zephyr Z9（轻薄便携手机）。#目的# 生成微博推广帖（≤500字），引导点击购买。#风格# 参考头部数码品牌广告文案风格。#语气# 有说服力。#受众# 年轻数码爱好者。#输出# 简洁、有传播力的微博文案。 |
| 输出（简化示例） | 内容泛化、缺少细节与目标受众指向。 | 围绕轻薄、高性能、续航与影像卖点展开，句式简洁、可传播，含行动召唤与品牌信任背书，更贴合目标受众。 |

## 优化 Prompt

### 技巧一：为模型提供输出样例

- 作用：让模型“模仿”格式、语气、文法与概念，提升一致性与稳定性。

| 项 | 未添加样例 | 添加样例 |
| --- | --- | --- |
| Prompt | #背景# 生成一篇小红书种草笔记，推广强森吹风机（体积小、高颜值、风力大、干得快、智能控温）。#受众# 追求时尚的年轻女性。#输出# 小红书文章格式，充满 emoji，简洁但充实。 | 在左侧基础上增加“#语气与风格#”模块：提供 5 种高点击风格公式（亲测体验、问题-原因-解法、独到见解、亲身经历、成果展示），并示例句式，要求正文按其风格展开。 |
| 输出（简化示例） | 格式与风格不稳定、表达泛化。 | 输出结构稳定、风格明确：标题+场景+卖点+情感收尾，语气自然，行动召唤清晰。 |

### 技巧二：设定完成任务的步骤

对于复杂任务，显式给出步骤可显著提升正确率。

数学题示例 Prompt（通过添加`#任务步骤#`）：

```plaintext
#背景#
又到了周日，小明像以往一样去爷爷奶奶家玩，小明于早上八点整步行出发，已知小明每分钟行走50米，
走了12分钟后，小明的父亲发现小明忘记带作业了，于是便骑车去追小明，已知小明的爸爸每分钟骑行200米，
等到追上小明后，爸爸决定骑车带上小明，已知小明坐自行车的路程是走路路程的5倍，
#目的#
小明什么时候到爷爷家？
#任务步骤#
1. 计算爸爸追上小明的时间与距离。
2. 计算剩余路程与需要时间。
3. 计算最终到达时间。
```

输出（简化）包含：

```plaintext
追上发生在出发后第16分钟（8:16），步行总距离800米；剩余路程4000米；骑行速度200米/分，需20分钟；总耗时36分钟，8:36到达。
```

### 技巧三：使用分隔符号区分单元

- 选择罕见且明显的分隔符（如：`###`、`===`、`>>>`）。
- 目的：提升模型对复杂 Prompt 的解析能力，防止内容混淆。

| 项 | 未使用分隔符 | 使用分隔符 |
| --- | --- | --- |
| Prompt | 请简短总结以下影评（长段落）。 | 请简短总结以下影评。### 段落1 ### 段落2 ### 段落3 ### |
| 输出（简化示例） | 单段总结，结构松散。 | 分段总结，结构清晰、要点对应分隔片段。 |

### 技巧四：引导模型“思考”

对于逻辑推理与语境学习的复杂任务，可通过**思维链（Chain of Thought, CoT）**与**提示链（Prompt Chaining）**显式引导推理过程。

任务目的：解析一个复杂的 JSON 结构体

```json
#背景#
JSON 输入：
{"web-app": {
  "servlet": [
    {
      "servlet-name": "cofaxEmail",
      "servlet-class": "org.cofax.cds.EmailServlet",
      "init-param": {"mailHost": "mail1", "mailHostOverride": "mail2"}},
    {
      "servlet-name": "cofaxTools",
      "servlet-class": "org.cofax.cms.CofaxToolsServlet",
      "init-param": {
        "templatePath": "toolstemplates/",
        "log": 1,
        "logLocation": "/usr/local/tomcat/logs/CofaxTools.log",
        "logMaxSize": ""}}],
  "servlet-mapping": {"cofaxEmail": "/cofaxutil/aemail/*", "cofaxTools": "/tools/*"},
  "taglib": {"taglib-uri": "cofax.tld", "taglib-location": "/WEB-INF/tlds/cofax.tld"}}}

#目的#
判断是否满足：
1) 每个 servlet 都有 init-param；
2) mapping 元素都在 servlet 中有匹配；
3) cofaxTools 有 3 个以 "log" 开头的参数，且 "log" < 10。
```

| 项 | 不使用思维链 | 使用思维链 |
| --- | --- |
| 输出（简化示例） | 直接结论：不符合/符合。 | 先逐条分析三项，再给出结论；可解释 `logMaxSize` 的值空导致的判定歧义。 |

#### 提示链（Prompt Chaining）引导样例

第一轮输入（简化）：

```plaintext
#目的# 计算被爸爸追上时刻与走过距离。
```

输出1（简化）：

```plaintext
8:16 被追上，总路程800米。
```

第二轮输入：

```plaintext
#目的# 计算小明坐自行车路程。
```

输出2（简化）：

```plaintext
4000 米。
```

第三轮输入：

```plaintext
#目的# 计算到达时间。
```

输出3（简化）：

```plaintext
总耗时36分钟；8:36到达。
```

## Prompt 测试与迭代

- 生成最优 Prompt 需要迭代试验与持续优化。
- 优化流程：设定目标 → 设计结构 → 提供样例 → 引导思考/步骤 → 线上反馈 → 调整修正。
- 用户反馈是提升模型输出质量的关键，应纳入闭环。

## Prompt 优化案例：跨国公司 AI 助手多语言稳定输出

问题背景：用户反馈某模型在英文问题上不能稳定用英文回答。

优化点：

1. 用“语种”替换“语言”，避免歧义。
2. 用 Prompt 框架重排结构；将“限制”拆分到对应位置。
3. 使用分隔符将 `${documents}` 独立为重要内容块。

| 项 | 优化前 Prompt（摘要） | 优化后 Prompt（摘要） |
| --- | --- | --- |
| 结构 | 角色、技能、工具与限制混杂，语种与标点约束不集中。 | #背景# 跨国公司 HR AI 助手；====== `${documents}` ====== #目的# 三类问题（政策解析/考勤答疑/年假咨询）处理标准；知识库不足时转人工。 |
| 约束 | 要求使用 ASCII、按用户语种回复、限定问题范围、隐私保护等，分散在“限制”中。 | #多语言要求# 非中文问题用中文含义检索；输出转换为用户语种。#输出# 仅使用标准 ASCII；回复语种与用户一致。 |
